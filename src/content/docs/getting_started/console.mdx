The console is a combination of a [component](/getting_started/ecs/components)
and a [system](/getting_started/ecs/systems) from WasserXR's core library.

It gives you runtime access to [ECS](/getting_started/ecs) functions. You can
manipulate components, load [plugins](/getting_started/ecs/plugins), .... It is
the clostest that currently exists to an **editor**.

## Instantiation

<Tabs syncKey="lang">
<TabItem label="C">

You can instantiate the console by running

```c
WXR_Entity console = wxr_add_entity(scene);
wxr_add_component(scene, console, "WXR_Console");

wxr_add_system(scene, "wxr_console_system", 100);
```

</TabItem>
</Tabs>

## Usage

When using the console, you have direct access to it through the standard input
of your system. This means that you can when running your application start
typing commands into the console.

### Loading and Saving Scenes

You can load and save your scene by running in the console

```
save <path_to_scene_file>
```

<Aside type="caution" title="Paths" icon="warning">

When using the `save` command all subdirectories in the path need to exist.
Otherwise, the command will fail.

</Aside>

```
load <path_to_scene_file>
```

### Hot Reloading

You can trigger a hot reload by running

```
reload
```

### Exit

You can exit the application by running

```
exit
```

### Entity Management

You can add or delete [entities](/getting_started/ecs/entities)

```
addEntity
```

This will return the identifier for the [entity](/getting_started/ecs/entities)

```
removeEntity <id>
```

### Component Management

You can add and delete [components](/getting_started/ecs/components)

```
addComponent <entity_id> <component_name>
```

```
removeComponent <entity_id> <component_name>
```

### Accessing Components

You can access [component](/getting_started/ecs/components) that are defined in
the
[component schema](/getting_started/ecs/components#step-4-implement-the-schema-function)

```
get <entity_id> <component_name> <field_name>
```

```
set <entity_id> <component_name> <field_name> <value>
```

<Aside type="note" title="Value encoding" icon="information">

The type hint that is defined in the
[component schema](/getting_started/ecs/components#step-4-implement-the-schema-function)
defines how the value will be encoded to show it in the console or how it is
expected to be delivered to the console during the `set`.

For example, `WXR_L` will encode everything as a `long`. `WXR_F` will treat the
data as a float encoding.

</Aside>

### System Management

You can add and delete [systems](/getting_started/ecs/systems)

```
addSystem <system_name>
```

```
removeSystem <system_name>
```

### Plugin Management

You can load or unload [plugins](/getting_started/ecs/plugins)

```
loadPlugin <path_to_plugin>
```

```
unloadPlugin <path_to_plugin>
```

### Inspecting Scene

You can inspect the what is currently in the scene by running

```
showEntities
```

```
showPlugins
```

```
showSystems
```

```
showComponents
```
