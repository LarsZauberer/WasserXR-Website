<Aside type="caution" title="Platform Support" icon="warning">
  WasserXR has only been tested on **Linux**. It has never been tested on
  Windows or macOS and will very likely not work on those platforms at the
  moment.
</Aside>

WasserXR consists of two parts: **WasserXR**, the foundational ECS, and
**WasserXR-Core**, the standard library of
[components](/getting_started/ecs/components),
[systems](/getting_started/ecs/systems), and
asset types.

<Tabs syncKey="lang">
<TabItem label="Rust">

To build a Rust project with WasserXR, first initialize a Rust project with

```bash
cargo init
cargo add wasserxr
```

In `main.rs`, add the following content:

```rust
use wasserxr::scene::{logging::file_logger, Scene};

fn main() {
    let mut scene = Scene::new();
    scene.register_callback_logger(file_logger);

    scene
        .load_plugin("./libwasserxr_core.so".to_owned())
        .expect("Failed to load core plugin");

    scene
        .add_system("console".to_owned(), 100)
        .expect("Failed to add system `console`");

    while scene.tick() {}
    let _ = scene.reset();
}
```

Before running the project, download the latest
[WasserXR-Core Plugin](https://github.com/LarsZauberer/WasserXR-Core/tags)
version from GitHub and put the `.so` file for Linux in your project directory.

When running this with

```bash
cargo run
```

you should see a terminal user interface (TUI). In the next chapters, we will
see what you can do with it.
