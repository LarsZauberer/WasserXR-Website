WasserXR implements it's own logging system.

## Usage

<Tabs syncKey="lang">
<TabItem label="Rust">

After creating a scene, the logging system has automatically been initialized
and you can start logging using the following macros

```rust
debug!(scene, "My Log: {}", my_variable_to_format);
info!(scene, "My Log: {}", my_variable_to_format);
warn!(scene, "My Log: {}", my_variable_to_format);
error!(scene, "My Log: {}", my_variable_to_format);
```

</TabItem>
</Tabs>

## Accessing the Log Ring

<Tabs syncKey="lang">
<TabItem label="Rust">

To access the whole log ring, you can call

```rust
for entry in scene.iter_log() {
  ...
}
```

</TabItem>
</Tabs>

## Callback Log Handler

There are log handler functions that will run every time a new log entry is
added to the log ring.

There are the following default log handlers already implemented in WasserXR:

- `print_logger` (Prints the logging output to the stdout (to the terminal))
- `file_logger` (Appends to the file `log.log`)

<Tabs syncKey="lang">
<TabItem label="Rust">

You can register these log handlers by running

```rust
scene.register_callback_logger(file_logger)
```

</TabItem>
</Tabs>

## Custom Log Handler

<Tabs syncKey="lang">
<TabItem label="Rust">

You can easily implement your own log handler. A log handler is just a function
that takes a reference to a `&LogEntry`.

For example the `print_logger` is implemented like this

```rust
pub fn print_logger(entry: &LogEntry) {
    println!("{}", entry);
}
```

</TabItem>
</Tabs>
