The logging system is a system (**not an ECS system**) directly built into the
`wasserxr_ecs` library.

It should help the user get important information about the
[ECS](/getting_started/ecs) and also about maybe problems that occur in a
[system](/getting_started/ecs/systems)/[component](/getting_started/ecs/components).

## Usage

<Tabs syncKey="lang">
<TabItem label="C">

To initialize the logging you can do the following

```c
#include "WasserXR/ecs/logging.h"

int main() {
  // Create logging
  wxr_logging_init(WXR_LOG_DEBUG);
  wxr_add_logger(wxr_stdout_logger);

  wxr_info("Logging Initialized!");
}
```

You can use the following methods to send a logging message.

```c
wxr_debug(...)
wxr_info(...)
wxr_warn(...)
wxr_error(...)
wxr_critical(...)
```

<Aside type="caution" title="Critical Level" icon="warning">

The critical logging level should only be used for unrecoverable crashes. It
will also directly exit the application afterwards.

</Aside>

<Aside type="note" title="Formatting" icon="information">

You can also use the same syntax like for `printf` with the functions to format
values into the string.

For example

```c
int x = 5;

wxr_debug("Value of x: %d", x);
```

</Aside>

</TabItem>
</Tabs>

## Asserts

<Tabs syncKey="lang">
<TabItem label="C">

The logging system also provides integrated assert statements.

```c
int x = 1;

wxr_assert(x == 1, "x is not 1. It is %d", x); # Triggers a critical log and quits the application
wxr_assert_recoverable(x == 1, "x is not 1. It is %d", x); # Triggers an error log but doesn't fail the application
wxr_assert_abort(x == 1, "x is not 1. It is %d", x); # Triggers an error log and calls a `return;` statement to end the current function
wxr_assert_abort(x == 1, -1, "x is not 1. It is %d", x); # Triggers an error log and calls a `return -1;` statement to end the current function
```

</TabItem>
</Tabs>

## Custom Log Handler

<Tabs syncKey="lang">
<TabItem label="C">

You can use your custom log handler. You need to define some kind of function
with the following signature

```c
void wxr_my_logger(WXR_Log_Entry entry);
```

The `WXR_Log_Entry` struct looks like this

```c
typedef struct {
  WXR_Log_Level level;
  char *msg;
} WXR_Log_Entry;
```

The `WXR_Log_Level` are an enum that looks like this

```c
typedef enum {
  WXR_LOG_DEBUG,
  WXR_LOG_INFO,
  WXR_LOG_WARN,
  WXR_LOG_ERROR,
  WXR_LOG_CRITICAL,
} WXR_Log_Level;
```

<Aside type="note" title="Ownership" icon="information">

The message in the WXR_Log_Entry is not owned by the logger. You shouldn't free
it.

</Aside>

To register your logger, you can do the following

```c
wxr_add_logger(wxr_my_logger);
```

where you pass in the function pointer.

</TabItem>
</Tabs>
