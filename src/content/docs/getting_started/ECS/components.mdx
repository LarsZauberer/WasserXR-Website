Components are pure data containers in ECS. They hold data for the
[entities](/getting_started/ecs/entities) but contain no logic themselves (this
is the task of [systems](/getting_started/ecs/systems)). Think of components as
the attributes that define what an [entity](/getting_started/ecs/entities) is and
what it can do.

## Structure of a Component

To make a component work with WasserXR's ECS, you need to provide three
functions:

- **Creator**: Allocates and initializes a new component instance (basically,
  the **constructor**)
- **Destroyer**: Frees a component instance and cleans up resources (basically,
  the **destructor**)
- **Schema**: Defines the component's fields so systems can access them without
  knowing about the Rust struct directly. It is also used to serialize and
  deserialize the component.

Luckily there are nice macros to help you define those functions easily.

## Writing Your First Component

<Tabs syncKey="lang">
<TabItem label="Rust">

To create a component, you can write something like this:

```rust
#[component]
#[derive(Default)]
struct Health {
  #[mutable]
  current_health: usize,
  max_health: usize
}

#[component_creator(Health)]
fn create_health(_scene: &mut Scene) -> Option<Health> {
  Some(Health::default())
}
```

<Aside type="note" title="Name of the component" icon="information">

The name of the component is determined by the name of the struct. In this case,
it is `"Health"`

</Aside>

</TabItem>
</Tabs>

## Creating and Destroying Components

Components cannot exist without an [entity](/getting_started/ecs/entities).
Every component lives on an [entity](/getting_started/ecs/entities) and cannot
outlive it.

When you destroy an [entity](/getting_started/ecs/entities), all the
components associated with it will be destroyed.

<Tabs syncKey="lang">
<TabItem label="Rust">

To instantiate a component, you can run:

```rust
let _ = scene.add_component(entity_id, "Health".to_owned());
```

To destroy a component, you can run:

```rust
let _ = scene.remove_component(entity_id, "Health");
```

</TabItem>
</Tabs>

<Aside type="caution" title="Same components on the same entity" icon="warning">

An [entity](/getting_started/ecs/entities) can only have each component type
instantiated once on it.

If you try to instantiate `"Health"` on the same entity again, it will return an
error.

</Aside>

## Accessing Fields

<Tabs syncKey="lang">
<TabItem label="Rust">

In Rust, you can access component fields by getting immutable or mutable
references with the corresponding functions:

```rust
// Immutable references only
let query_result: Result<(&usize, &usize), SceneError> = scene.query::<(&usize, &usize)>(entity_id, "Health", &["current_health", "max_health"]);

// Mutable and immutable references
let query_mut_result: Result<(&mut usize, &usize), SceneError> = scene.query_mut::<(&mut usize, &usize)>(entity_id, "Health", &["current_health", "max_health"]);
```

</TabItem>
</Tabs>
