Entities are the simplest concept in an ECS. An entity is an ID that groups
[components](/getting_started/ecs/components). An entity that has no components
has no behavior or data beyond its ID and name.

<Aside type="note" title="Example" icon="list-format">

For example, an entity can be your player. The player might have transform,
health, model, and controller components. There might be other entities with
different combinations of [components](/getting_started/ecs/components).

</Aside>

An entity is represented by its
[Uuid](https://en.wikipedia.org/wiki/Universally_unique_identifier) which is
basically a 16-byte value.

An entity also has a name. Names have no semantic meaning, but they help the
developer distinguish between different entities.

## Creating Entities

<Tabs syncKey="lang">
<TabItem label="Rust">

To create an entity you can call

```rust
let entity: Uuid = scene.add_entity();
```

</TabItem>
</Tabs>

## Naming Entities

<Tabs syncKey="lang">
<TabItem label="Rust">

You can name an entity by calling

```rust
scene.set_entity_name(entity_id, "My Entity".to_owned());
```

You can query the name again by calling

```rust
let name_res: Result<&str, SceneError> = scene.get_entity_name(entity_id);
```

</TabItem>
</Tabs>

## Destroying Entities

<Tabs syncKey="lang">
<TabItem label="Rust">

You can simply destroy an entity by calling

```rust
let _ = scene.remove_entity(entity_id);
```

</TabItem>
</Tabs>

<Aside type="caution" title="Components on entities" icon="warning">

When an entity is destroyed, it will automatically remove all the components
associated with it.

</Aside>
