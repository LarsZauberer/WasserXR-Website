Entities are the simplest concept of an ECS. It is a container for
[components](/getting_started/ecs/components). An entity that has no components
has no meaning.

<Aside type="note" title="Example" icon="list-format">
  For example, an entity can be your player. The player might have a transform,
  health, model, controller component. There might be other entities with
  different combinations of [components](/getting_started/ecs/components).
</Aside>

An entity is saved in the type `WXR_Entity`. This is the identifier to reference
the entity that we might want to manipulate.

## Creating Entities

<Tabs syncKey="lang">
<TabItem label="C">

To create a new entity, use `wxr_add_entity()`:

```c
WXR_Scene *scene = wxr_create_scene();

// Create some entities
WXR_Entity player = wxr_add_entity(scene);
WXR_Entity enemy = wxr_add_entity(scene);
WXR_Entity camera = wxr_add_entity(scene);
```

</TabItem>
</Tabs>

## Destroying Entities

<Tabs syncKey="lang">
<TabItem label="C">

```c
WXR_Entity player = wxr_add_entity(scene);
int result = wxr_remove_entity(scene, player);
```

This removes the entity and **automatically destroys all its components**. The
entity ID becomes invalid after removal.

</TabItem>
</Tabs>

## Getting All Entities

<Tabs syncKey="lang">
<TabItem label="C">

Sometimes you need to iterate over all entities in a scene:

```c
size_t count;
WXR_Entity *entities = wxr_get_entities(&count, scene);

printf("Scene has %zu entities\n", count);
for (size_t i = 0; i < count; i++) {
    printf("Entity: %zu\n", entities[i]);
}

free(entities);  // Remember to free!
```

<Aside type="note" title="Ownership" icon="information">
  The array returned by `wxr_get_entities()` is owned by you and must be freed.
</Aside>

</TabItem>
</Tabs>
