A resource is also not a classical ECS concept.

In WasserXR a resource is some simple data that is persistent throughout the
entirety of the application run. It won't be hotreloaded.

<Aside type="caution" title="When to use" icon="warning">

Be careful when using the resource. Do not overuse it. It goes against the
concept modularity and hotreloadability in WasserXR.

Still, it is sometimes not possible to avoid or you sometimes don't want certain
things to hotreload.

</Aside>

## Usage

<Tabs syncKey="lang">
<TabItem label="Rust">

You can simply create a resource by running

```rust
let _ = scene.add_resource::<usize>("my_resource_key_name".to_owned(), 5);
```

To access the resource you can use the following functions

```rust
let res: Result<&usize, SceneError> = scene.get_resource::<usize>("my_resource_key_name");
let res: Result<&mut usize, SceneError> = scene.get_mut_resource::<usize>("my_resource_key_name");
```

</TabItem>
</Tabs>

<Aside type="caution" title="Resource Naming" icon="warning">

The naming of resources is global and not bound to any plugin. Be careful when
naming resources to not interfer with resources from other plugins.

</Aside>
