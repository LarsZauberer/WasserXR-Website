Assets are also not a necessarily standard ECS object. In WasserXR assets are
pretty similar to [components](/getting_started/ecs/components) but have a lot
of differences.

An asset is a global object that stores information. This information is cached
and immutable. Every asset has an asset type, which defines how to construct a
certain asset from a data string. There can be multiple assets creates from a
single asset type.

<Aside type="note" title="Example: Model" icon="open-book">

In the WasserXR-Core plugin, there exists a `ModelAsset`. This asset type
interprets the given data string is a file path and loads the whole Model.

Other systems can after loading a Model read the model data, like their
vertices.

</Aside>

## Usage

<Tabs syncKey="lang">
<TabItem label="Rust">

Using a asset is quite simple and works almost the same as a
[component](/getting_started/ecs/components)

There are mainly two steps. First, you need to ensure that the asset is properly
loaded and then you can query data from it.

```rust
let res: Result<(), SceneError> = scene.ensure_asset_loaded(asset_type, data_string);
let res: Result<(&usize,), SceneError> = scene.asset_query_loaded::<(&usize,)>(asset_type, data_string, &["my_field"]);
```

When you need a shorthand for both of these operations together, you can also
run

```rust
let res: Result<(&usize,), SceneError> = scene.asset_query::<(&usize,)>(asset_type, data_string, &["my_field"]);
```

</TabItem>
</Tabs>

<Aside type="caution" title="Reloading" icon="warning">

A hotreload will invalidate the asset cache and they will be reinitialized.

</Aside>

## Creating your own Asset Type

<Tabs syncKey="lang">
<TabItem label="Rust">

Creating your own asset type, works also pretty much the same as building your
own [component](/getting_started/ecs/components).

```rust
#[asset_type]
struct MyAsset {
  ...
}

#[asset_type_creator(MyAsset)]
fn create_my_asset(scene: &mut Scene, data_string: &str) -> Option<MyAsset> {
  ...
}
```

You can also use the `#[none]` field attribute to make a field not queryable.

</TabItem>
</Tabs>
