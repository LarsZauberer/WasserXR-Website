# Opening the WasserXR Blog

WasserXR now has a dedicated blog area next to the documentation. The docs will
continue to explain how to install, learn, and use the engine, while the blog
will cover the conversations around the engine: what changed, why certain design
choices were made, and where community feedback would be especially useful.

<Aside type="tip" title="Built for people already watching the engine" icon="star">

If you already know the basics of WasserXR, the blog is intended to be the
lighter-weight place to keep up with development without rereading setup guides
or API references.

</Aside>

## Why add a blog?

A project like WasserXR has two very different communication needs:

- **Documentation** should stay practical, stable, and task-focused.
- **Blog posts** can be more conversational, timely, and transparent about
  current development work.

That separation keeps legal, licensing, and feature information prominent on the
main page while still giving the community a clear path to updates.

## Planned topics

Future posts can include:

- progress updates for ECS, hot reload, and XR integration work;
- technical notes about implementation trade-offs;
- demo announcements and community testing requests;
- migration notes when an early-access feature changes shape.

## Follow along

Start with the docs if you are new to WasserXR. Visit the blog when you want to
understand where the engine is heading and how to get involved earlier in the
process.
